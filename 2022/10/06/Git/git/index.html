# GIT

# 一、git 特性

  1. 直接记录快照，而非差异比较

    • git 快照：在原有文件版本的基础之上重新生成一份新的文件 (备份)
    • 效率：未做修改的文件则不会重新存储，而是保留一个链接指向之前存储的文件

    缺点： 占用磁盘空间较大

    优点： 版本切换时非常快，因为每个版本都是完整的文件快照

  2. 近乎所有操作都是本地执行

    特性：

    1. 断网后依旧可以在本地进行版本控制

    2. 联网后，将本地记录同步到云端服务器即可

# 二、git 中的三个区域

  1. 工作区
  2. 暂存区
  3. git 仓库

# 三、三种状态

  1. 已修改 (modified)
    • 表示已经修改了文件，但是还没有将修改结果放入暂存区
  2. 已暂存 (staged)
    • 表示对已修改文件的当前版本做了标记，使之包含于下次提交的列表中
  3. 已提交 (committed)
    • 表示当前文件已经保存在本地的 git 仓库中

  • 在当前文件夹下右键点击 Git bash

  1. modified（文件已被修改，暂未放入暂存区）

  1. double modified

# 4. 基础命令

    • git reset HEAD 目标文件
    • git reset HEAD . 移除所有暂存区中文件

  4. 跳过使用暂存区

    • git commit -a -m "描述信息"
    • 自动把所有跟踪过的文件暂存起来一并提交

  1. 推送

    第一次将本地分支推送到远程仓库
# -u 表示把本地分支和远程分支进行关联
# 只有第一次推送需要带 -u 参数
git push -u 远程仓库别名 本地分支名称:远程分支名称
如果希望本地分支和远程分支名称保持一致
git push -u origin 名称
查看远程仓库所有分支信息
git remote show origin

# 将远程仓库下载到本地仓库
git checkout 远程分支名称
# 从远程仓库中，把对应的远程分支下载到本地仓库，并把下载的本地分支进行重命名
git checkout -b 本地分支名称 远程仓库/远程分支
git checkout -b master origin/main

# 拉取远程分支最新代码
git pull

# 删除远程分支
git push 远程仓库名 --delete 远程分支名

# 查看远程分支
git branch -r
git branch -a

# 从远程获取代码库
git fetch
