# Http 超文本通信协议

a: 序号（sequence number）： > Seq 序号，占 32 位，用来标识从 TCP 源端向目的端发送的字节流，发起方发送数据时对此进行标记。

对序号的补充: > 应用层向 TCP 层发送用于网间传输的、用 8 位字节表示的数据流，然后 TCP 把数据流分区成适当长度的 报文段 （通常受该计算机连接的网络的 数据链路层 的 最大传输单元 （ MTU ）的限制）。之后 TCP 把结果包传给 IP 层，由它来通过网络将包传送给接收端实体的 TCP 层 **。TCP 为了保证不发生丢包，就给每个包一个序号，同时序号也保证了传送到接收端实体的包的按序接收。** 然后接收端实体对已成功收到的包发回一个相应的确认（ ACK ）；如果发送端实体在合理的往返时延（RTT）内未收到确认，那么对应的数据包就被假设为已丢失将会被进行重传。TCP 用一个 校验和 函数来检验数据是否有错误；在发送和接收时都要计算校验和。

b: 确认序号（acknowledgement number）： > Ack 序号，占 32 位，只有 ACK 标志位为 1 时，确认序号字段才有效，Ack=Seq+1。

c: 标志位（Flags）： > 共 6 个，即 URG、ACK、PSH、RST、SYN、FIN 等。具体含义如下 > URG：紧急指针（urgent pointer）有效。 > ACK：确认序号有效，确认接收到消息； > PSH：接收方应该尽快将这个报文交给应用层。 > RST：重置连接。 > SYN：发起一个新连接。 > FIN：释放一个连接。

# TCP 三次握手

  1. ** 客户端向服务器端发送 ** SYN = 1 报文，并初始化序列号 seq = x , 此时客户端处于 SYN_SENT 阶段，等待服务器端确认请求
  2. ** 服务器端收到包后，根据标志位 ** SYN = 1 表示知道服务器端请求连接，同时服务器端将同步标志位 SYN 和确认标志位 ACK 置为 1，同时 ack = seq + 1 ，并初始化序列号 seq = y ，服务器端进入 SYN_RCVD 状态
  3. ** 客户端收到服务器端的确认包后，发送一个 ** ACK 请求后， ACK = 1``ack = seq + 1 , 此时客户端处于 ESTABLISHED 状态，服务器收到 ACK 报文后，也处于 ESTABLISHED 状态，此时双方建立连接

# 浏览器同源策略

  1. ** 相同通信协议 ** http https
  2. 相同域名
  3. 相同端口号

# http 的工作流程

  1. 和服务器建立连接 ==> tcp 的三次握手
  2. 客户端向服务器发送请求 (request)
  3. 服务器端收到客户端发来的请求进行相应处理后做出回应
  4. 断开与服务器的连接 ==> 四次挥手

# http 的完整工作流程

  1. 进行域名解析
  2. 发起 tcp 的三次握手
  3. 建立连接后向服务器发送请求
  4. 服务器端接收到请求后响应请求，浏览器接收到 html 代码
  5. 浏览器对 html 代码进行解析，并向服务器请求 html 中的资源
  6. 浏览器对页面进行渲染

# http 响应状态码（status）

分类描述
1**信息，服务器收到请求，需要请求者继续执行操作
2**成功，操作被成功处理
3**重定向，需要进一步操作完成请求
4**客户端错误，请求包含语法错误或无法完成请求
5**服务器错误，服务器在处理请求的过程中发生错误

200：请求成功。

301：被请求的资源已永久移动到新位置。

**304：** 请求资源未被修改，浏览器

401：当前请求需要用户验证。

403：服务器已经理解请求，但是拒绝执行它。

404：服务器未找到资源，网页不存在

500：服务器遇到了一个未曾预料的状况，导致了它无法完成对请求的处理。一般来说，这个问题都会在服务器的程序码出错时出现。

# get 请求

  • 安全性低
  • 参数以 querystring 的形式发送
  • get 请求会被浏览器主动缓存
  • get 请求根据浏览器的不同长度也不同

# post 请求

  • post 传输的数据放在 body 里面
  • 请求不会被浏览器主动缓存，除非手动设置
  • post 请求没有限制，如果服务端没有限制
  • 对参数类型没有限制，理论上可以传输任何数据类型，但要和请求头对应
  • 传输的数据类型一般为 json 类型

# get 和 post 的区别？

  • post 比 get 安全
  • get 请求比 post 请求快
  • get 传输内容有限制，post 理论上没有
  • get 请求参数放在请求路径上 (?参数=值&参数=值) ，post 放在请求体中
  • get 请求是字符串，post 可以是任何数据类型

# readyState 属性用来表示当前 ajax 请求状态

0 xhr 对象已经创建，尚未调用 open 方法
1 open () 方法以备调用
2 send () 调用，响应头也已被接收
3 数据接收中，response 属性中已经包含部分数据
4 ajax 请求完成

# 状态码 status

# 2** 成功

  1. 200 请求成功 get post
  2. 201 已创建。成功请求创建了资源 post put

# 3** 重定向

  1. 301 永久移动 请求的资源被永久移动到新的 url
  2. 302 临时移动 类似 301 但资源只是临时移动 客户端继续使用原有 url
  3. 304 未修改 不返回任何资源，客户端通常会访问缓存资源

# 4** 客户端错误

  1. 404 网页不存在
  2. 400 请求错误
  3. 401 当前请求需要用户验证
  4. 403 服务器拒绝访问
  5. 408 请求超时

# 5** 服务器错误

  1. 500
  2. 501 服务器不支持请求方法
  3. 503 超载或系统维护
